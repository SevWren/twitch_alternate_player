ROOT CAUSE (Simple Explanation):
--------------------------------

JavaScript code is looking for HTML elements with Russian IDs:
  - "статистика-сжатиевидео" (video compression)
  - "статистика-сервер" (server)

These elements were renamed to English IDs in the HTML file:
  - "statistics-videocompression"
  - "statistics-server"

But the JavaScript code was NOT updated, so it's looking for elements that
no longer exist. When it can't find them, it crashes.

================================================================================

WHAT HAPPENS:
-------------

1. Error reporting code runs (player.js:730-760)
2. Tries to collect statistics data for the report (line 743)
3. Statistics module reads DOM elements to get current values (line 1841-1859)
4. Line 1846: Tries to access "статистика-сжатиевидео"
5. Element doesn't exist, returns null
6. Узел() function tries to check null.nodeType
7. TypeError: Cannot read property 'nodeType' of null
8. Crash during crash reporting!

WHY THIS MATTERS:
-----------------

This bug breaks the error reporting system. When the player encounters an
error and tries to generate a crash report, it crashes while generating the
report itself. This means:

  - No crash reports are sent to developers
  - Bugs go unreported
  - Issues are harder to diagnose
  - User experience degrades silently

Fixing this ensures that when real bugs occur, developers get proper
diagnostic information to fix them.

This project is undergoing Russian → English translation. The maintainer does
not speak Russian. This bug occurred because HTML translation was incomplete:

  ✅ HTML element IDs were changed to English (partially)
  ❌ JavaScript references were NOT updated
  ❌ Changes were not done atomically

The proper translation process should:
  1. Update HTML IDs and JavaScript references in same commit
  2. Test thoroughly before committing
  3. Follow atomic commit principle (no partial translations)

This bug is a learning example for the translation effort going forward.
